# 한국형 SaaS ERP 시스템 개발 계획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**K-ERP SaaS Platform** - 한국 중소기업을 위한 클라우드 기반 통합 경영관리 시스템

### 1.2 프로젝트 목표
- 국세청 전자세금계산서 연동을 포함한 완전한 한국형 회계 시스템 구축
- 4대보험(국민연금, 건강보험, 고용보험, 산재보험) EDI 연동
- 급여/인사관리 시스템 통합
- 멀티테넌트 SaaS 아키텍처로 확장 가능한 플랫폼 구축

### 1.3 대상 고객
- **Primary**: 연매출 10억~100억 중소기업
- **Secondary**: 스타트업 및 소규모 사업자
- **Enterprise**: 중견기업 (향후 확장)

### 1.4 핵심 가치 제안
| 기존 ERP 문제점 | K-ERP 솔루션 |
|----------------|-------------|
| 높은 초기 도입 비용 | 월 구독 모델, 초기 비용 없음 |
| 복잡한 설치/유지보수 | 클라우드 기반, 자동 업데이트 |
| 정부 연동 별도 구매 | 세금계산서/4대보험 기본 포함 |
| 경직된 커스터마이징 | 모듈형 구조, 유연한 확장 |

### 3.2 기술 선정 근거

| 영역 | 기술 | 선정 이유 |
|------|------|----------|
| **Backend** | Go 1.22+ (Gin) | 고성능, 동시성 처리, 정부 API 연동 |
| **ORM** | GORM + sqlc | CRUD + 복잡 쿼리 최적화 |
| **Frontend** | React 18 + TypeScript + Vite | 컴포넌트 재사용, 타입 안정성 |
| **UI Library** | shadcn/ui + Tailwind CSS | 일관된 디자인, 빠른 개발 |
| **Database** | PostgreSQL 16 | JSON 지원, 한글 전문검색 |
| **Cache** | Redis 7 | 세션, 캐시, 실시간 데이터 |
| **Message Queue** | NATS JetStream | 비동기 처리, Go 친화적, 고성능 |
| **Container** | Docker + K8s | 확장성, 배포 자동화 |

## 4. 보안 요구사항

### 4.1 데이터 보안
- **암호화**: AES-256 (저장 데이터), TLS 1.3 (전송 데이터)
- **키 관리**: AWS KMS 또는 HashiCorp Vault
- **접근 제어**: RBAC (역할 기반 접근 제어)

### 4.2 인증/인가
- **인증**: OAuth 2.0 + JWT
- **MFA**: TOTP 기반 2단계 인증
- **SSO**: SAML 2.0 지원 (기업 고객용)

### 4.3 컴플라이언스
- **개인정보보호법** 준수
- **전자금융거래법** 준수
- **ISMS 인증** 준비 (향후)
