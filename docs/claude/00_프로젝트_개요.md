## 1. 프로젝트 소개

### 1.1 프로젝트명

**K-ERP SaaS Platform** - 한국형 클라우드 ERP 시스템

### 1.2 프로젝트 배경

한국 중소기업(연매출 10억~1,000억원)을 위한 클라우드 기반 통합 ERP 시스템입니다. 기존 On-Premise ERP의 높은 초기 비용과 유지보수 부담을 해소하고, 한국 비즈니스 환경에 최적화된 기능을 제공합니다.

### 1.3 주요 차별점

| 항목 | 기존 ERP | K-ERP SaaS |
|------|----------|------------|
| 초기 비용 | 수천만원~수억원 | 월 구독료 (인당 5~15만원) |
| 구축 기간 | 6개월~2년 | 즉시 사용 가능 |
| 정부기관 연동 | 수동 또는 별도 비용 | 기본 내장 (홈택스, 4대보험) |
| 세금계산서 | 별도 ASP 계약 필요 | 무료 스크래핑 + 유료 API 선택 |
| 업데이트 | 추가 비용 발생 | 자동 업데이트 포함 |

### 1.4 버전 히스토리

| 버전 | 날짜 | 변경 내용 |
|------|------|----------|
| v0.1 | 2026-01-15 | 초기 설계 (Go 단일 언어) |
| **v0.2** | 2026-01-16 | **Go + Python 하이브리드 아키텍처 전환** |

### 1.5 v0.2 주요 변경사항

v0.1에서 v0.2로 전환하면서 다음 사항이 변경되었습니다:

1. **언어 아키텍처 변경**: Go 단일 언어 → Go + Python 하이브리드
2. **마이크로서비스 분리**: 암호화/스크래핑 기능을 Python 서비스로 분리
3. **gRPC 통신 도입**: 서비스 간 고성능 통신 프로토콜 적용
4. **비용 최적화**: 세금계산서 무료 스크래핑 우선 전략

```
우선순위 1: 홈택스 스크래핑 (무료, Python 서비스)
    ↓ 실패 시
우선순위 2: 국세청 API (무료, Go 서비스)
    ↓ 실패 시
우선순위 3: Popbill ASP (유료, Go 서비스)
```

**예상 비용 절감 효과**:
- 월 10,000건 발행 기준
- 스크래핑 성공률 80% 가정
- 연간 약 4,800만원 절감 (건당 500원 기준)

## 3. 기술 아키텍처 개요

### 3.1 하이브리드 아키텍처 도입 배경

#### 3.1.1 Go 언어의 한계

한국 정부기관 연동에는 SEED-CBC, ARIA-CBC 등 한국 표준 암호화 알고리즘이 필수입니다. Go 언어의 표준 라이브러리와 생태계에서는 이러한 알고리즘에 대한 검증된 라이브러리가 부재합니다.

| 암호화 알고리즘 | 사용처 | Go 지원 | Python 지원 |
|----------------|--------|---------|-------------|
| SEED-CBC | 홈택스, 국민연금, 고용보험, 산재보험 | 미지원 | pycryptodome, kisa-seed |
| ARIA-CBC | 건강보험공단 | 미지원 | pycryptodome |
| PKCS#7 서명 | 4대보험 EDI | 제한적 | cryptography |

#### 3.1.2 Python 도입 이점

1. **검증된 암호화 라이브러리**: pycryptodome, cryptography
2. **풍부한 스크래핑 라이브러리**: Selenium, Playwright, BeautifulSoup
3. **기존 홈택스 라이브러리**: hometax (PyPI)
4. **개발 기간 단축**: 3주 이상 → 1주 이내
